## 2. Design tokens

- ฟอนต์: Anuphan (Google Fonts) ทุกหน้า
- ไม่ใช้ emoji ทุกหน้า — ใช้ไอคอนเส้นล้วน (prototype ใช้ inline SVG; production ควรใช้ชุดไอคอนมาตรฐานชุดเดียว เช่น Tabler ให้สเกล/น้ำหนักเส้นเท่ากัน)
- การ์ด: พื้นขาว radius 16–22px เงาบาง
- โทนปฏิเสธ/อันตราย: `#d8513f` พื้น `#fdecea`
- **ธีมสีเปลี่ยนได้** ผ่าน CSS variables 4 ตัว (ดู §7): `--bubble` (สีหลัก), `--green` (accent), `--green-soft` (พื้นอ่อน), `--app-bg` (พื้นแอป) — ดู §7

## 3. Flow onboarding (ลำดับล่าสุด)

1. **ตั้งชื่อ** — ถามผ่าน chat input จริง (พิมพ์แล้วกดส่ง ไม่ใช้การ์ดฟอร์ม) แยก 2 จังหวะ: ชื่อผู้ใช้ → ชื่อผู้ช่วย
2. **ลองตั้งเตือน + เลือกโทน** — แตะตัวอย่าง (ใช้เคสในชีวิตประจำวัน เช่น "เตือนรดน้ำต้นไม้พรุ่งนี้เช้า 7 โมง" ไม่ใช้เคสงาน) → เห็นการ์ดผลลัพธ์ → เลือกโทนจาก **การ์ดตัวอย่างประโยคจริง** (ครับ/ค่ะ/จ๊ะ/ไม่ใส่)
3. **คำเรียก** — ถามทุกคน ตัวเลือกปรับตามโทน (ดู §4.2) + กำหนดเอง
4. **สรุปไฟล์** — ใช้ **ไฟล์ตัวอย่าง** (ป้ายกำกับ "ไฟล์ตัวอย่าง" ชัดเจน ไม่ปลอมว่าเป็นไฟล์ผู้ใช้) ผลลัพธ์แสดงเป็น **carousel การ์ดเลื่อนแนวนอน** (เช่นทริป 3 วัน = 3 การ์ด) — *production: ขั้นนี้ควรให้ผู้ใช้เลือก/อัปโหลดไฟล์เอง หรือกดข้าม*
5. **คุยด้วยเสียง** — แนะนำให้กดปุ่มไมค์ค้างเพื่อพูด + ประโยคตัวอย่าง (เช่น "ตั้งข่าวทุกวันตอนแปดโมง") — *production: ต่อ speech-to-text จริง; demo จำลองการถอดเสียง*
6. **เลือกธีมสี** — ขั้นสุดท้าย เลือกสีแล้วทั้งแอปเปลี่ยนทันที (ดู §7)

**สำคัญ: persona เริ่มจาก neutral** ทุกข้อความก่อนผู้ใช้เลือกโทนต้องไม่มีคำลงท้าย (ไม่ default เป็น ค่ะ) แล้วค่อยใช้คำลงท้ายตามที่เลือกตั้งแต่ขั้นคำเรียกเป็นต้นไป

## 4. โมเดลบุคลิก

### 4.1 น้ำเสียง (tone) — default = neutral
| tone | บอกเล่า | คำถาม |
|---|---|---|
| male | ครับ | ครับ |
| female | ค่ะ | คะ |
| casual | จ๊ะ | จ๊ะ |
| neutral | (ไม่ใส่) | (ไม่ใส่) |

### 4.2 คำเรียกผู้ใช้ (address) + บทบาทผู้ช่วย (self-reference)
ตัวเลือกคำเรียกขึ้นกับ tone:
- casual: `[ชื่อ]`, แก, นาย, เพื่อน
- neutral: `[ชื่อ]`, คุณ`[ชื่อ]`
- polite: `[ชื่อ]`, คุณ`[ชื่อ]`, พี่`[ชื่อ]`, น้อง`[ชื่อ]`

**คำเรียกบอกบทบาทผู้ช่วยด้วย** (ฟังก์ชัน `selfName()`):
- เรียกผู้ใช้ขึ้นต้น "พี่…" → ผู้ช่วยเป็นน้อง → แทนตัว "น้อง`[ชื่อผู้ช่วย]`"
- เรียกผู้ใช้ขึ้นต้น "น้อง…" → ผู้ช่วยเป็นพี่ → แทนตัว "พี่`[ชื่อผู้ช่วย]`"
- อื่น ๆ → แทนตัวด้วยชื่อปกติ

ในหน้า settings: self-reference เป็นฟิลด์แยก ตั้งอัตโนมัติตามคำเรียกแต่ override เองได้ (ชื่อ / พี่+ชื่อ / น้อง+ชื่อ / หนู)

### 4.3 กฎคำลงท้าย (continuity) — ต้องทำให้ถูก
- **คำลงท้ายติดกับคำหน้า ไม่มีช่องว่าง**: "เรียบร้อยจ๊ะ" ไม่ใช่ "เรียบร้อย จ๊ะ"
- **หลัง "นะ / สิ / ล่ะ" เพศหญิงใช้รูป "คะ"** (เสียงสูง) ไม่ใช่ "ค่ะ": "ตลอดนะคะ" ไม่ใช่ "ตลอดนะค่ะ" — กฎง่าย ๆ: ถ้าประโยคจบด้วย นะ/สิ/ล่ะ ให้ใช้รูปคำถาม (คะ)
- neutral ต้องไม่มีคำลงท้ายทุกชนิดให้สม่ำเสมอ

### 4.4 พรีวิวสด (settings)
recompute ทุกครั้งที่เปลี่ยนค่า รูปแบบ: `[selfDisplay]ตั้งเตือนให้[addressUser]แล้ว[particle]`
เช่น casual + "น้องบอล" + "พี่ปิ่น" → "พี่ปิ่นตั้งเตือนให้น้องบอลแล้วจ๊ะ"

## 5. บุคลิกพิเศษ (ชั้น opt-in)

แยกจาก onboarding หลัก เข้าจากปุ่มในหน้า settings เป็นคู่สรรพนามผูกความสัมพันธ์:

| key | เรียกผู้ใช้ | แทนตัวเอง | 18+ |
|---|---|---|---|
| friend | แก / นาย | เรา / ชื่อ | ไม่ |
| butler | นายท่าน | กระหม่อม / บ่าว | ใช่ |
| mom | ลูก / หนู | แม่ | ใช่ |
| cute | ตัวเอง / ที่รัก | เค้า / ชื่อ | ใช่ |
| custom | (กรอกเอง) | (กรอกเอง) | ขึ้นกับเนื้อหา |

- คู่ 18+ ต้องผ่านการ์ดยืนยัน (bottom sheet): สรุปคำเรียก/แทนตัว/น้ำเสียง + หมายเหตุยังเป็นผู้ช่วยทำงานเหมือนเดิม + checkbox ยืนยันอายุ ปุ่มเปิดใช้ disabled จนกว่าติ๊ก
- ขอบเขต: เปลี่ยนแค่คำเรียก+น้ำเสียง prompt ต้องตรึง role "ผู้ช่วย" เสมอ
- มีแถบ "กำลังใช้ …" + ปุ่ม "กลับเป็นปกติ" เด่นเสมอ + ป้ายสถานะ

## 6. ตรวจ input / ปฏิเสธชื่อกวน

กรองทั้ง **ชื่อผู้ช่วย / ชื่อผู้ใช้ / คำเรียก / คำแทนตัว** เพราะถูกผู้ช่วยพูดออกเสียงทุกข้อความ (ดู `validate()` / `normalize()` ใน `pin-reject-name-ux.html`)

ลำดับตรวจ:
1. ว่าง หรือไม่มีตัวอักษรไทย/อังกฤษเลย → `symbol`
2. ยาวเกิน ~14 ตัวอักษร → `long`
3. `normalize()` (ตัดเว้นวรรค, เลขแทนอักษร, ตัวซ้ำรัว→2) แล้วเทียบ INJECT (admin/system/ระบบ/ignore/ลืมคำสั่ง) → `inject`
4. เทียบ BLOCK (หยาบ/เหยียด/ลามก ไทย+อังกฤษ+สแลง) → `profane`
5. ผ่าน → รับค่า

UX: ปฏิเสธแบบนุ่มอยู่ในคาแรกเตอร์, **ห้ามผู้ช่วยพูดคำแย่ ๆ ซ้ำ** (mask บับเบิล), ขอบแดง + shake + ตัวนับอักษร, พยายามเกิน 3 ครั้งเสนอ "ข้ามไปก่อน ใช้ชื่อจริง"

**บังคับสองชั้น: server ต้องกรองซ้ำเสมอ** (client bypass ได้) ลิสต์ใน prototype เป็นตัวอย่าง ของจริงต้องครอบคลุมและ maintain ต่อเนื่อง

## 9. Acceptance / QA

- ก่อนเลือกโทน ทุกข้อความเป็นกลาง (ไม่มีคำลงท้าย); หลังเลือกเปลี่ยนทันที
- female สลับ ค่ะ/คะ ถูกตามประโยค และหลัง นะ/สิ/ล่ะ ใช้ "คะ"
- คำลงท้ายติดคำหน้าไม่มีช่องว่าง
- เลือก "น้อง[ชื่อ]" → ผู้ช่วยแทนตัวเป็น "พี่[ชื่อ]" อัตโนมัติ; override self-reference แล้วไม่ถูก reset ทับ
- พรีวิว settings ตรงกับข้อความจริง
- สรุปไฟล์แสดงเป็น carousel เลื่อน/snap ได้; ไฟล์มีป้าย "ไฟล์ตัวอย่าง"
- กดไมค์ค้าง → สถานะ "กำลังฟัง"; ปล่อย → ถอดเสียง + ทำงาน
- เลือกธีม → ทั้งแอปเปลี่ยนสีทันที; รีเซ็ต/เริ่มใหม่กลับ default
- persona 18+ เปิดไม่ได้ถ้าไม่ติ๊กอายุ; "กลับเป็นปกติ" รีเซ็ตจริง
- คำหยาบ/ยาว/สัญลักษณ์/คำสั่งระบบ ในทุกช่อง input → ปฏิเสธด้วย reason ถูก ทั้ง client+server; ผู้ช่วยไม่พูดคำต้องห้ามซ้ำ

## 10. คำถามค้างถึง product

- เจ้าของและการ maintain ลิสต์คำต้องห้าม (ไทย/อังกฤษ/สแลง)
- นโยบายแอบอ้างบุคคล/แบรนด์ในชื่อ — บล็อก/เตือน/ปล่อย
- การยืนยัน 18+ ใช้ checkbox หรือผูกกับบัญชี
- ถ้าผู้ใช้เป็นเยาวชน ต้องปิดบุคลิกพิเศษกลุ่มโรแมนติก/ครอบครัวทั้งหมดไหม
- ขั้นสรุปไฟล์ใน production: ให้เลือก/อัปโหลดไฟล์เอง + ปุ่มข้าม
