## 3. โมเดลบุคลิก (logic หลักที่ต้อง implement)

### 3.1 น้ำเสียง (tone)
ตัวเลือกในการ์ด ผูกค่าคำลงท้ายตามนี้:

| tone | บอกเล่า | คำถาม |
|---|---|---|
| male | ครับ | ครับ |
| female | ค่ะ | คะ |
| casual | จ๊ะ | จ๊ะ |
| neutral | (ไม่ใส่) | (ไม่ใส่) |

หมายเหตุ: female ต้องสลับ ค่ะ↔คะ ตามชนิดประโยค; neutral ต้องไม่มีคำลงท้ายทุกชนิด

### 3.2 คำเรียกผู้ใช้ (address) + บทบาทของผู้ช่วย (self-reference)
ตัวเลือกคำเรียกขึ้นกับ tone:
- casual: `[ชื่อ]`, แก, นาย, เพื่อน
- neutral: `[ชื่อ]`, คุณ`[ชื่อ]`
- polite: `[ชื่อ]`, คุณ`[ชื่อ]`, พี่`[ชื่อ]`, น้อง`[ชื่อ]`

**สำคัญ: คำเรียกบอกบทบาทของผู้ช่วยด้วย** (logic จาก `selfName()` ใน prototype)
- เรียกผู้ใช้ขึ้นต้น "พี่…" → ผู้ช่วยเป็นน้อง → แทนตัว "น้อง`[ชื่อผู้ช่วย]`"
- เรียกผู้ใช้ขึ้นต้น "น้อง…" → ผู้ช่วยเป็นพี่ → แทนตัว "พี่`[ชื่อผู้ช่วย]`"
- อื่น ๆ → แทนตัวด้วยชื่อปกติ

ในหน้า settings ให้ self-reference เป็นฟิลด์แยกที่ตั้งค่าอัตโนมัติตามคำเรียก แต่ override เองได้ (ตัวเลือก: ชื่อ / พี่+ชื่อ / น้อง+ชื่อ / หนู)

### 3.3 พรีวิวสด (settings)
ทุกการเปลี่ยนค่าให้ recompute ประโยคตัวอย่าง รูปแบบ:
`[selfDisplay]ตั้งเตือนให้[addressUser]แล้ว[particle]`
เช่น tone=casual + เรียก "น้องบอล" + แทนตัว "พี่ปิ่น" → "พี่ปิ่นตั้งเตือนให้น้องบอลแล้วจ๊ะ"

## 4. บุคลิกพิเศษ (ชั้น opt-in)

แยกจาก onboarding หลัก เข้าจากปุ่มในหน้า settings เป็นคู่สรรพนามผูกความสัมพันธ์:

| key | เรียกผู้ใช้ | แทนตัวเอง | 18+ |
|---|---|---|---|
| friend | แก / นาย | เรา / ชื่อ | ไม่ |
| butler | นายท่าน | กระหม่อม / บ่าว | ใช่ |
| mom | ลูก / หนู | แม่ | ใช่ |
| cute | ตัวเอง / ที่รัก | เค้า / ชื่อ | ใช่ |
| custom | (กรอกเอง) | (กรอกเอง) | ขึ้นกับเนื้อหา |

กติกา:
- คู่ที่ 18+ ต้องผ่านการ์ดยืนยัน (bottom sheet): สรุปคำเรียก/แทนตัว/น้ำเสียง + หมายเหตุว่ายังเป็นผู้ช่วยทำงานเหมือนเดิม + checkbox ยืนยันอายุ 18+ ปุ่มเปิดใช้ disabled จนกว่าจะติ๊ก
- ขอบเขตบทบาท: เปลี่ยนแค่คำเรียก+น้ำเสียง ระบบ prompt ต้องตรึง role "ผู้ช่วย" ไม่ให้บทบาทลากบทสนทนาออกนอกขอบเขต
- มีแถบ "กำลังใช้ …" + ปุ่ม "กลับเป็นปกติ" เด่นเสมอ
- เก็บ `persona_mode` แยก และโชว์ป้ายสถานะว่ากำลังใช้บุคลิกพิเศษ

## 5. ตรวจ input / ปฏิเสธชื่อกวน (สำคัญ)

เพราะชื่อ/คำเรียกถูกผู้ช่วย "พูดออกเสียงทุกข้อความ" ต้องกรองทั้ง **ชื่อผู้ช่วย / ชื่อผู้ใช้ / คำเรียก / คำแทนตัว** (ดู `validate()` + `normalize()` ใน `pin-reject-name-ux.html`)

ขั้นตอนตรวจ:
1. trim แล้วถ้าว่าง หรือไม่มีตัวอักษรไทย/อังกฤษเลย → reason `symbol`
2. ยาวเกิน ~14 ตัวอักษร → reason `long`
3. `normalize()` (ตัดเว้นวรรค, เลขแทนอักษร 0→o/1→i/3→e/4→a, ตัวซ้ำรัว→2) แล้วเทียบ
4. ตรงลิสต์ INJECT (admin, system, ระบบ, ignore, ลืมคำสั่ง, …) → reason `inject`
5. ตรงลิสต์ BLOCK (คำหยาบ/เหยียด/ลามก ไทย+อังกฤษ+สแลง) → reason `profane`
6. ผ่าน → รับค่า

หลักการ UX:
- ปฏิเสธแบบนุ่ม อยู่ในคาแรกเตอร์ ไม่ดุ ไม่สั่งสอน วนกลับไปถามใหม่
- **ห้ามให้ผู้ช่วยพูดคำแย่ ๆ ซ้ำ** (mask บับเบิลคำไม่เหมาะสม)
- ฟีดแบ็กที่ช่อง: ขอบแดง + shake + ตัวนับอักษร
- พยายามเกิน 3 ครั้ง → เสนอทางออก "ข้ามไปก่อน ใช้ชื่อจริง"

copy ปฏิเสธ (อ้างอิงได้):
- profane: "คำนี้ปิ่นเรียกแล้วไม่ค่อยน่ารักเลย ขอเปลี่ยนเป็นคำอื่นได้ไหมคะ"
- long: "ขอสั้น ๆ สัก 1–2 คำได้ไหมคะ"
- symbol: "พิมพ์เป็นตัวอักษรให้ปิ่นหน่อยนะคะ"
- inject: "คำนี้ใช้ตั้งไม่ได้นะคะ ลองคำอื่นดูไหมคะ"

**บังคับสองชั้น:** client กรองเพื่อ UX ทันที แต่ **server ต้องกรองซ้ำเสมอ** (client bypass ได้) ลิสต์ใน prototype เป็นตัวอย่าง ของจริงต้องทำลิสต์ไทย/อังกฤษ/สแลงให้ครบและ maintain ต่อเนื่อง

## 6. Data model (เสนอ)

| ฟิลด์ | ตัวอย่าง | หมายเหตุ |
|---|---|---|
| user_name | "บอล" | ผ่าน validate |
| assistant_name | "ปิ่น" | ผ่าน validate |
| tone | male / female / casual / neutral | |
| address_user | "น้องบอล" | ผ่าน validate |
| self_reference | "พี่ปิ่น" | auto จาก address แต่ override ได้ |
| persona_mode | basic / special:butler / custom | |
| age_confirmed | true / false | จำเป็นเมื่อ persona 18+ |
| custom_pair | {call, self} | เมื่อ persona = custom |

## 7. Acceptance / QA checklist

- เปลี่ยน tone กลางคัน → ทุกข้อความถัดไปเปลี่ยนคำลงท้ายทันที, female สลับ ค่ะ/คะ ถูกตามประโยค
- เลือก "น้อง[ชื่อ]" → ผู้ช่วยแทนตัวเป็น "พี่[ชื่อ]" อัตโนมัติ และข้อความปิดท้าย onboarding สะท้อนถูก
- override self-reference แล้วไม่ถูกคำเรียก reset ทับ
- พรีวิวใน settings ตรงกับข้อความจริงที่ผู้ช่วยพูด
- persona 18+ เปิดไม่ได้ถ้าไม่ติ๊กยืนยันอายุ; "กลับเป็นปกติ" รีเซ็ตจริง
- ใส่คำหยาบ/ยาวเกิน/สัญลักษณ์/คำสั่งระบบ ในทุกช่อง input → ถูกปฏิเสธด้วย reason ถูกต้อง ทั้ง client และ server
- ผู้ช่วยไม่พูดคำต้องห้ามซ้ำในข้อความปฏิเสธ
- เปิดบุคลิกพิเศษค้างไว้ → ป้ายสถานะยังเห็น เข้าถึงปุ่มรีเซ็ตได้ตลอด

## 8. คำถามค้างถึง product

- ลิสต์คำต้องห้ามฉบับจริง (ไทย/อังกฤษ/สแลง) ใครเป็นเจ้าของและ maintain
- นโยบายแอบอ้างบุคคล/แบรนด์ ในชื่อ — บล็อก เตือน หรือปล่อย
- เกณฑ์อายุ/การยืนยัน 18+ ใช้แค่ checkbox หรือผูกกับบัญชี
- ถ้าผู้ใช้เป็นเยาวชน ต้องปิดบุคลิกพิเศษกลุ่มโรแมนติก/ครอบครัวทั้งหมดไหม
